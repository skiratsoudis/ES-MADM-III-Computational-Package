# ES-MADM III Decision Studio

This folder contains the final R/Shiny implementation of the ES-MADM III computational model.

## File

```text
ES_MADM_III_Decision_Studio_Final.R
```

## Purpose

The ES-MADM III Decision Studio implements the computational workflow used to generate the manuscript results. It provides an interactive environment for importing scenario-specific Excel input files, executing the ES-MADM III calculations, reviewing the resulting weights, scores, rankings, entropy measures, and diagnostic indices, and exporting consolidated computational outputs.

The application corresponds to the finalized ES-MADM III formulation used in the manuscript and excludes earlier development-stage versions.

---

## Implemented Computational Layers

The application supports the following computational layers:

1. Import of structured Excel input files.
2. Definition of alternatives, criteria, criterion type, fuzzy performance values, fuzzy subjective weights, and preference-function parameters.
3. Ratio-based normalization for benefit and cost criteria.
4. PROMETHEE-type preference conditioning through pairwise performance gaps.
5. Construction of preference-enhanced conditional probability structures.
6. Entropy-based evaluation of criterion informativeness.
7. Construction of feasible operational representatives.
8. Derivation of information-sensitive representative objective and integrated weights.
9. Computation of final alternative scores and rankings.
10. Computation of Integrated Criteria Importance.
11. Computation of system-level entropy and information measures.
12. Computation of diagnostic indices, including NMI, CES, ADI, and scenario-relative NMGI.
13. Export of consolidated computational results.

---

## Recommended Workflow

The recommended workflow for reproducing the manuscript results is:

1. Open `ES_MADM_III_Decision_Studio_Final.R` in RStudio.
2. Run the Shiny application.
3. Import the S1 baseline input file through the main data import interface.
4. Import S2, S3, S4A, and S4B through the scenario portfolio workflow.
5. Review the scenario-level computational outputs.
6. Export the consolidated results workbook.
7. Compare the exported workbook with the reference output file stored in the `results/` folder.

The five input files required for the manuscript case study are stored in the `data/` folder of this repository.

## Output Generated by the Application

The application generates computational outputs including:

- representative subjective, objective, and integrated weights;
- operational entropy values;
- operational diversification degrees;
- Integrated Criteria Importance values;
- final alternative scores;
- final rankings;
- system-level entropy measures;
- mutual information values;
- diagnostic indices;
- scenario-relative NMGI synthesis;
- benchmark-supporting outputs;
- consolidated Excel export files.

The reference output workbook generated from the final application version is stored in:

```text
results/ES_MADM_III_Results_Computational_Outputs.xlsx
```

## Notes on Scenario Processing

For the manuscript case study, the baseline scenario is imported first through the main data import interface. The remaining scenarios are then imported through the scenario portfolio workflow.

This structure allows the application to produce both scenario-specific outputs and comparative multi-scenario results. The exported workbook should therefore be interpreted as the consolidated computational output of the full scenario family.

## Reproducibility Statement

This application is the final corrected R/Shiny implementation used for the manuscript results. It is intended for computational reproducibility, reviewer inspection, and transparent verification of the reported tables and diagnostic outputs.

Earlier scripts, obsolete model versions, superseded output files, and internal development materials are intentionally excluded from this repository.
